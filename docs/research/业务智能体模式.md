# 业务智能体模式

## 工作区优先

每个任务都拥有独立工作区。用户、后端、OpenCode 和业务工具都围绕这个工作区协作。

```text
run_xxx/
  input/
  sources/
  notes/
  output/
  logs/
```

## 可追溯交付

调研、分析、写作类智能体不应只输出结论，还应该保存来源、摘要、中间分析和最终交付物。

## 人工审批

外部副作用、敏感文件访问、不可逆操作都应该经过用户确认。

## 技能沉淀

反复出现的任务流程应沉淀为：

- Agent profile
- Command
- Custom tool
- MCP connector
- 输出模板
