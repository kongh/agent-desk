# 事件流设计

## 目标

- 将 OpenCode 原始事件转成用户可读的任务时间线。
- 保留原始事件用于调试。
- 将低层事件聚合成业务里程碑。

## 时间线示例

- 正在理解任务
- 正在制定计划
- 正在搜索资料
- 正在阅读来源
- 正在保存证据
- 正在撰写报告
- 等待权限审批
- 最终输出已生成

## 设计注意

不要把所有底层事件原样暴露给用户。产品层应该对事件进行清洗、分组和命名。

## 当前 SDK 映射

`OpenCodeSdkRunner` 已接入 `client.global.event()`，将 OpenCode 原始事件映射为任务时间线事件。

当前映射范围：

- `session.created` -> `opencode-session`
- `session.status` -> `opencode-status`
- `session.idle` -> `opencode-idle`
- `session.error` -> `failed`
- `message.updated` -> `opencode-message`
- `message.part.updated` 中的 tool/text/step -> `tool`、`opencode-message`、`opencode-step`
- `file.edited` -> `artifact`
- `permission.updated` -> `permission`

过滤规则：

- 只接收当前 `workspace.path` 对应的 OpenCode global event。
- 有 `sessionID` 的事件必须匹配当前任务创建出的 `session.id`。
- 每条映射后的事件保留 `rawType`，便于后续调试和扩展。
