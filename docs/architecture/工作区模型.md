# 工作区模型

每个任务拥有一个独立工作区：

```text
run_xxx/
  input/
  sources/
  notes/
  output/
  logs/
```

## 目录语义

- `input/`：用户上传文件、初始任务上下文。
- `sources/`：搜索结果、网页内容、引用材料、原始证据。
- `notes/`：中间分析、草稿、结构化摘要。
- `output/`：最终交付物，例如 `report.md`、`slides.md`、`deck.pptx`。
- `logs/`：执行日志、事件快照、调试信息。

## 设计原则

- 工作区是智能体的操作台。
- 最终结果必须能从工作区里找到。
- 关键中间过程应可审计。
